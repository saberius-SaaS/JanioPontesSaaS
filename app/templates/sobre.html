DOCUMENTO CRÍTICO E RESTRITO A ADMINISTRADORES

Este documento contém informações sensíveis e essenciais para a continuidade do Janio Pontes SaaS. Em caso de ausência permanente ou incapacidade do idealizador e gestor principal (Janio Pontes), este documento deve ser entregue a um Engenheiro de Software ou Desenvolvedor Full-Stack Sênior para assumir a manutenção e evolução do sistema.

1. Visão Geral do Sistema

O Janio Pontes SaaS é a plataforma de gestão interna e portal do cliente da Janio Pontes Contabilidade. Ele gerencia tarefas, protocolos, automação de WhatsApp, e-mails e área do cliente.

  • URL do Sistema: app.janiopontes.com.br
  • URL do Chatwoot (WhatsApp): chat.janiopontes.com.br

2. Tecnologias Utilizadas (Tech Stack)

Um novo desenvolvedor precisará destas tecnologias para dar continuidade ao projeto:

Linguagem & Backend Python 3.12, FastAPI, Uvicorn
Banco de Dados PostgreSQL (SQLAlchemy / Alembic)
Frontend HTML (Jinja2), Tailwind CSS v4, HTMX
Automação IA Estruturado em IA Agent (Pasta .agents)

3. Infraestrutura de Produção (GCP)

O sistema está inteiramente hospedado no ecossistema do Google Cloud Platform (GCP).

  • Projeto GCP: jp-saas-producao
  • Google Cloud Run: Serviço jp-saas-app (us-east1)
  • Google Cloud SQL: Instância jpsaas-db-us (us-east1)
  • Google Cloud Storage (GCS): Bucket janio-pontes-saas-docs-us
  • Firebase Hosting: Conectado ao Cloud Run para gerenciar o domínio.
  • Cloud Scheduler (Cronjobs): Região southamerica-east1 para alertas e relatórios.

4. Acessos, Credenciais e Co-Propriedade

Para evitar pontos únicos de falha, a infraestrutura deve sempre possuir mais de uma conta administrativa (Co-Propriedade). Os acessos fundamentais são:

  • Administrador e Proprietário (Owner) no Google Cloud (IAM) do projeto jp-saas-producao. Estão configurados como proprietários o e-mail principal (janiopontes@janiopontes.com.br) e o e-mail de contingência oficial (tecnologia@janiopontes.com.br).
  • Acesso Técnico/Administrativo ao Registro.br (domínio janiopontes.com.br). O acesso de contingência está vinculado ao ID JAPTE25.
  • Acesso Root (Administrador) na plataforma do Chatwoot (utilizando a conta tecnologia@janiopontes.com.br).
  • Acesso ao provedor onde o Código Fonte está hospedado (Google Drive / Repositório Git).

Deploy & Segredos

As variáveis de ambiente estão configuradas diretamente no Cloud Run: JPSAAS_DB_PASSWORD, JPSAAS_SECRET_KEY, JPSAAS_GOOGLE_CLIENT_SECRET, JPSAAS_GCP_CREDENTIALS.

Comando padrão de publicação (via terminal na pasta do projeto):
gcloud run deploy --source .
